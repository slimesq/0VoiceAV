# 0VoiceAVLearning

Audio/video learning project using CMake and Conan.

## Dependencies

This project uses FFmpeg from Conan Center:

- `ffmpeg/7.1.3`
- Static libraries
- GPL-related dependencies such as x264/x265 enabled through Conan options

Conan Center does not publish an exact `ffmpeg/7.1` recipe, so the project pins
the latest available 7.1 patch release: `7.1.3`.

## Build

If VS Code clangd reports `STL1000: Unexpected compiler version`, update LLVM
clangd to the version expected by your Visual Studio STL, or keep the provided
`.clangd` file, which suppresses that editor-only version check.

Install dependencies. Press Enter at both prompts to use the defaults:
`light` flavor and `Debug` build type.

```powershell
.\pull_dependency.ps1
```

You can also skip the prompts:

```powershell
.\pull_dependency.ps1 -Flavor light -BuildType Debug
.\pull_dependency.ps1 -Flavor full -BuildType Release
```

Conan builds use all detected logical CPU cores by default. Limit parallelism if
the machine gets too busy:

```powershell
.\pull_dependency.ps1 -Flavor full -BuildType Release -Jobs 8
```

After a successful install, repeating the same flavor/build type skips
`conan install` and reuses the generated files under `build/generators`. Force a
fresh dependency graph only when you changed Conan options or want to refresh
packages:

```powershell
.\pull_dependency.ps1 -Flavor light -BuildType Debug -ForceConan
```

The `full` flavor may compile FFmpeg locally. After switching flavors or build
types, reconfigure CMake before building. On Windows, `full` disables ZeroMQ
because the Conan Center FFmpeg recipe currently has trouble detecting libzmq
through pkg-config on the MSVC path.

When `light + Debug` is selected, the script installs Release Conan packages
and the CMake project maps Debug builds to those packages through
`VOICE_AV_USE_RELEASE_CONAN_DEPS`. This avoids compiling FFmpeg just to debug
this project. Turn that CMake option off only when you install matching Debug
Conan packages.

The script first searches `third_party` for an existing FFmpeg source tree. If
none is found, it downloads FFmpeg source code into `third_party/ffmpeg-src` for
editor navigation. The folder is ignored by git.
It also generates clangd navigation files inside that source tree, so jumping
from FFmpeg headers to matching `.c` implementations can work after clangd
finishes background indexing.
The merged clangd database is written to `build/clangd/compile_commands.json`
and copied to the ignored root `compile_commands.json` for editor compatibility.
This file is generated by `pull_dependency.ps1`; Visual Studio does not need to
generate `compile_commands.json`.
Dependencies are stored in a project-local Conan cache under `build/conan_home`
so this project is isolated from any broken global Conan cache.

Configure with CMake:

```powershell
cmake --preset voice-av-vs2022-x64
```

Build Debug or Release:

```powershell
cmake --build --preset voice-av-vs2022-x64-debug
cmake --build --preset voice-av-vs2022-x64-release
```

Open the generated Visual Studio solution:

```powershell
cmake --open build/cmake
```

Run:

```powershell
.\build\cmake\Debug\voice_av_learning.exe
.\build\cmake\Release\voice_av_learning.exe
```

## Lessons

Course examples live under `lessons`. The original `src` folder was renamed so
each example can be grouped by lesson.

- `lessons/00-ffmpeg-version`: prints the linked FFmpeg library versions.
- `lessons/08-ffmpeg-demuxing-and-decoding-practice`: imported Qt examples
  converted into auto-discovered CMake targets.

Build targets added for lesson 08:

```powershell
cmake --build --preset voice-av-vs2022-x64-debug --target lesson_08_05_decode_audio
cmake --build --preset voice-av-vs2022-x64-debug --target lesson_08_06_decode_video
```

In Visual Studio, set one of those lesson targets as the startup project and
run it. CMake configures the debugger working directory and default arguments
for the imported lesson 08 targets.

Run from PowerShell by passing paths explicitly:

```powershell
Push-Location .\lessons\08-ffmpeg-demuxing-and-decoding-practice\testFiles
..\..\..\build\cmake\Debug\lesson_08_05_decode_audio.exe believe.aac believe.pcm
..\..\..\build\cmake\Debug\lesson_08_06_decode_video.exe source.200kbps.768x320_10s.h264 source.yuv
Pop-Location
```

Lesson 08 input test files are kept together in
`lessons/08-ffmpeg-demuxing-and-decoding-practice/testFiles`.
The imported lesson 08 executables are written to the normal CMake build output
directory. Visual Studio debugging still uses `testFiles` as the working
directory with default arguments.

The old Qt project files and bundled FFmpeg 4.2.1 SDK from the archives were
not imported. This project now builds the examples through the root CMake
project and links against Conan's FFmpeg 7.1.3 package.

To add another lesson 08 example later, create a new subdirectory under
`lessons/08-ffmpeg-demuxing-and-decoding-practice` with a `main.c` or
`main.cpp`. CMake will create a target named from that folder, for example
`07-my-demo/main.c` becomes `lesson_08_07_my_demo`. Re-run configure after
adding the folder:

```powershell
cmake --preset voice-av-vs2022-x64
```

See `docs/ffmpeg-conan-options.md` for the requested native FFmpeg configure
flags that can and cannot be represented by the official Conan Center recipe.
